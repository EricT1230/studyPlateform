# 組合計數 (Combinatorics)

## 計數前先問三個問題

1. **有順序嗎？**（排列 vs 組合）
2. **可以重複嗎？**
3. **有「至少 / 至多」這種限制嗎？**

答對這三題，公式幾乎自動決定。

## 兩個基本原理

- **乘法原理**：一件事分成「接連的幾個獨立步驟」→ 各步選擇數**相乘**。
  例：3 件上衣 × 2 條褲子 = 6 種搭配。
- **加法原理**：分成「互斥的幾類情況」→ 各類數量**相加**。
  例：搭公車 4 條路線、捷運 2 條路線 → 共 6 種到法。

## 排列 vs 組合（最常混的）

從 5 個人選 3 個：

```
排列 (有順序，如選正/副/書記)：
  P(5,3) = 5 × 4 × 3 = 60

組合 (無順序，如選 3 人小組)：
  C(5,3) = P(5,3) / 3! = 60 / 6 = 10
```

差別就在「順序算不算」。組合 = 排列再除掉「同一組的內部排序」3!。

公式：
```
P(n,k) = n! / (n−k)!
C(n,k) = n! / (k!·(n−k)!)
```

## 重複物的排列

字母排列若有重複，要除掉重複物的階乘。
例：MISSISSIPPI（M1, I4, S4, P2，共 11 個）：

```
11! / (1!·4!·4!·2!)
```

## 排容原理 (Inclusion-Exclusion)

「至少符合一個」時，直接加會把重疊算兩次，要扣回交集：

```
|A ∪ B| = |A| + |B| − |A ∩ B|

三個：|A∪B∪C| = |A|+|B|+|C| − |A∩B|−|A∩C|−|B∩C| + |A∩B∩C|
```

「至少一個」常更好用**補集**：總數 − 「一個都不符合」。

## Stars and Bars：把 n 個分到 k 組

非負整數解 `x₁+x₂+…+x_k = n` 的個數：

```
C(n + k − 1, k − 1)
```

直覺：n 顆星星排成一列，插 k−1 個隔板分成 k 段。
例：把 5 顆糖分給 3 人（可有人 0 顆）→ C(5+2, 2) = C(7,2) = 21。

## 常見誤解

- 「選人組隊」是**組合**（無序），別當排列。
- 互斥情況才能直接相加；**有重疊**要用排容。
- 「至少一個」常用**補集**比逐一列舉快。
- 重複物排列**一定要除**掉相同物件的階乘。

## 解題時的判斷

- 先答三問：有序？可重複？有限制？
- 把「物件 / 位置 / 盒子」定義清楚，再決定要不要拆案例。
- 「至少」→ 想補集或排容。
- 「分配非負整數解」→ stars and bars。
- 寫出公式後，能不能說出「每個因子代表哪個選擇」？（能，通常就對了）
