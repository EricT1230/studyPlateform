# SQL 基礎

## 核心概念
SQL 是描述式查詢語言：你描述要哪些資料，資料庫決定如何取得。常見子句的邏輯順序是 `FROM`、`WHERE`、`GROUP BY`、`HAVING`、`SELECT`、`ORDER BY`，這能解釋別名、聚合與過濾的限制。

## 解題重點
JOIN 要分清 inner、left/right outer 與 self join，並確認連結條件是否會放大列數。`WHERE` 過濾列，`HAVING` 過濾分組。`COUNT(*)` 算列數，`COUNT(col)` 忽略該欄 NULL。`UNION` 去重，`UNION ALL` 保留重複。視窗函數可在不縮減列數下做排名或累計。外鍵維持參照完整性，唯一約束不等於一般索引提示。

## 常見陷阱
NULL 不是一般值，`= NULL` 不會成立，要用 `IS NULL`。`NOT IN` 若清單或子查詢含 NULL，可能得到意外空結果。字串拼接使用者輸入會造成 SQL injection，應用參數化查詢。`DISTINCT` 只影響輸出結果，不會清理原始資料。

## 練習前檢查
先標出資料來源、連結條件、分組欄位與聚合條件。若題目問結果列數，特別注意 NULL、重複列、outer join 補出的空值。若題目問語法位置，回到邏輯執行順序判斷。若題目出現子查詢，判斷它是否依賴外層列；相關子查詢可能逐列評估，效能與語意都不同。
