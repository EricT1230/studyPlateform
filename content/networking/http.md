# HTTP

## 這是什麼？

HTTP 是瀏覽器和伺服器之間的「對話規則」：你**問 (request)**、伺服器**答 (response)**。

```
 瀏覽器  ── request ───▶  伺服器
        ◀── response ──
```

最重要的特性：**無狀態 (stateless)**。伺服器預設**不記得**你上一次做了什麼，每個請求都是獨立的。
所以「登入、購物車」這種要記住的狀態，得靠額外機制（cookie / session / token）補。

## 一次請求和回應長怎樣

```
請求 (Request)                    回應 (Response)
┌──────────────────────┐        ┌──────────────────────┐
│ GET /users/1 HTTP/1.1│ 起始列 │ HTTP/1.1 200 OK       │
│ Host: example.com    │        │ Content-Type: ...     │ 標頭
│ Accept: ...          │ 標頭   │                       │
│                      │        │ { "id": 1, ... }      │ 主體
│ (GET 通常沒有主體)     │ 主體   │                       │
└──────────────────────┘        └──────────────────────┘
```

## 常見方法的語意

| 方法 | 用途 | 安全 (不改狀態) | 冪等 (重複=一次) |
|---|---|---|---|
| GET | 取得資源 | ✅ | ✅ |
| POST | 提交/建立 | ❌ | ❌ |
| PUT | 整體替換 | ❌ | ✅ |
| DELETE | 刪除 | ❌ | ✅ |

- **安全 (safe)**：不應改變伺服器狀態（GET）。
- **冪等 (idempotent)**：做 1 次和做 N 次，最終結果一樣（PUT 覆蓋成同值、DELETE 刪了再刪還是不存在）。POST 不冪等（送兩次可能建兩筆）。

## 狀態碼：看開頭數字就懂大方向

| 開頭 | 意思 | 例子 |
|---|---|---|
| 2xx | 成功 | 200 OK、201 Created |
| 3xx | 重新導向 | 301 永久、302 暫時 |
| 4xx | **你**（用戶端）錯 | 404 找不到、400 請求有誤 |
| 5xx | **伺服器**錯 | 500 內部錯誤、503 無法服務 |

高頻易混：**401 是「未驗證」**（還沒登入/憑證無效）；**403 是「已知身分但沒權限」**（再給憑證也沒用）。

## HTTPS：HTTP 穿上防護衣

HTTPS = HTTP 跑在 **TLS** 之上，提供三件事：
- **機密性**：加密，別人看不到內容。
- **完整性**：內容沒被竄改。
- **身分驗證**：透過憑證確認你連到的是真的那個網站。

## 版本演進（一句話記）

- **HTTP/1.1**：keep-alive 連線重用；但有隊頭阻塞。
- **HTTP/2**：一條連線上多工多個串流 + 壓縮標頭。
- **HTTP/3**：改用 **QUIC**（建在 UDP 上），消除 TCP 層隊頭阻塞、連線更快。

## 常見誤解

- **無狀態 ≠ 不能登入**：靠 cookie/token 在無狀態之上維持狀態。
- `Cache-Control: no-cache` **不是禁止儲存**，而是「用之前要再驗證」；`no-store` 才是完全不存。
- **CORS** 是瀏覽器的同源安全機制，不是伺服器之間的防火牆。

## 解題時的判斷

- 看到狀態題：先用開頭數字 (2/3/4/5) 定大方向，再分 401 vs 403。
- 方法題：先問「安全嗎？冪等嗎？」
- 「為什麼伺服器不記得我」→ 無狀態 + 該用 cookie/session/token。
