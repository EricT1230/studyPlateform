# DNS（網域名稱系統）

## 這是什麼？

人記得住 `google.com`，但電腦要的是 IP（像 `142.250.x.x`）。
DNS 就是那本「電話簿」：把好記的名字翻成機器要的 IP。

它是**分散式**的——沒有任何一台機器存全世界的名字，而是分層委派、各管一段。

## 名字的階層

網域名稱由右往左、一層層委派：

```
                     . (root 根)
                     │
        ┌────────────┼────────────┐
       com          org           tw       ← TLD（頂級網域）
        │
     example.com                            ← 權威名稱伺服器負責這一段
        │
   www.example.com                          ← 主機
```

讀法：`www.example.com.` 其實是「根 → com → example → www」。

## 一次查詢怎麼跑（解析流程）

你的電腦通常問一台 **遞迴解析器 (recursive resolver)**，它幫你一路問到底：

```
 你 ──「www.example.com 的 IP？」──▶ 解析器
                                      │
                                      │ ① 問 root：「.com 去問誰？」
                                      ▼
                                    root ──▶ 「去問 .com 伺服器」
                                      │ ② 問 .com：「example.com 去問誰？」
                                      ▼
                                   .com ──▶ 「去問 example.com 的權威伺服器」
                                      │ ③ 問權威伺服器：「www 的 IP？」
                                      ▼
                               權威伺服器 ──▶ 「142.250.x.x」
 你 ◀──── 回傳 IP ──────────────── 解析器
```

- 你對解析器是 **遞迴查詢**（要它負責拿到最終答案）。
- 解析器對各層是 **疊代查詢**（每次得到「再去問下一層」的指引）。

## 不是每次都從 root 問：快取

問過的答案會被**快取**（解析器、作業系統、瀏覽器都會存），下次直接用。
快取能存多久由 **TTL** 決定：
- TTL 長 → 查詢少、但改了 IP 要等比較久才生效。
- TTL 短 → 改動快生效、但查詢量大。換 IP 前常先調短 TTL。

## 常見紀錄類型

| 紀錄 | 用途 |
|---|---|
| A / AAAA | 名稱 → IPv4 / IPv6 |
| CNAME | 別名（指向另一個名稱） |
| MX | 這個網域的郵件伺服器 |
| NS | 這個網域由哪些權威伺服器負責 |
| PTR | 反查：IP → 名稱 |

## 傳輸與安全

- 一般查詢用 **UDP/53**（短小快）；回應太大被截斷、或做區域傳送 (zone transfer) 時改用 **TCP/53**。
- **DNSSEC**：對回應**數位簽章**，防偽造/竄改——但**不加密**查詢內容。
- **DoH / DoT**：把查詢**加密**傳輸，保護隱私。（和 DNSSEC 目的互補：一個防竄改、一個防偷看。）

## 常見誤解

- **DNSSEC 不加密**（只簽章驗證）；要加密是 DoH/DoT。
- **root 不存所有網域**，只負責「指引下一層去問誰」。
- **Anycast** 不是廣播給所有節點，而是「同一個 IP 由多地節點宣告」，路由把你導到最近的那台。
- 查不到/查到舊資料時，可能是**沿途快取**還留著舊答案（不一定是權威資料錯）。

## 解題時的判斷

- 解析題：先畫「你 → 解析器 → root → TLD → 權威」這條鏈。
- 遞迴 vs 疊代：誰負責「一路問到答案」。
- 安全題：要「防竄改」想 DNSSEC、要「防偷看」想 DoH/DoT。
