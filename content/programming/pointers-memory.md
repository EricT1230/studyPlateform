# 指標與記憶體

## 指標到底是什麼？

記憶體像一排有編號的格子，每個格子有**位址**。
**指標就是「存位址的變數」**——它不存資料本身，存的是「資料在哪裡」。

```
位址:   1000  1004  1008
值:     [ 42 ][ ?? ][ ?? ]
         ↑
   int *p = 1000;   // p 指向位址 1000 那格
   *p == 42         // 「去 p 指的地方拿值」
```

- `&x`：取得 x 的**位址**。
- `*p`：取用 p 指向位址上的**值**（dereference，解參考）。

```c
int x = 42;
int *p = &x;   // p 拿到 x 的位址
*p = 10;       // 透過 p 改 x → 現在 x == 10
```

## 為什麼需要指標？

1. 讓函式**改得到呼叫端的變數**（傳位址進去）。
2. 處理**動態大小**的資料（執行時才知道要多大）。
3. 高效傳遞大資料（傳位址，不複製整包）。

## 指標運算：以「指向型別大小」為單位

`p + 1` 不是位址加 1 byte，而是**跳過一個元素**。

```c
int a[10];
int *p = &a[2];
p + 3;   // 指向 a[5]，不是 a[2] 往後 3 byte
```

因為 int 佔 4 byte，`p+3` 實際位址前進 12 byte。這也是陣列 `a[i]` 等同 `*(a+i)` 的原因。

## 動態記憶體：自己借、自己還

```c
int *arr = malloc(n * sizeof(int));  // 跟系統借一塊
if (!arr) { /* 借不到要處理 */ }
// ... 使用 ...
free(arr);   // 用完歸還
arr = NULL;  // 避免之後誤用（懸空指標）
```

**誰借的誰負責還 (ownership)**。借了不還 → 記憶體洩漏；還兩次 → 破壞配置器。

## const 放哪裡，意思差很多

```c
const int *p;    // 不能透過 p 改「值」，但 p 可改指向
int * const p;   // p 不能改「指向」，但可透過 p 改值
```

口訣：const 緊貼誰，誰就被鎖住。

## realloc 要用暫時指標接

`realloc` 可能失敗回傳 NULL；若直接覆蓋原指標，會把原本那塊也弄丟。

```c
void *tmp = realloc(p, newSize);
if (tmp) p = tmp;     // 成功才換手
else { /* p 仍有效，處理失敗 */ }
```

## 改指標本身：要用「指標的指標」

若函式要改的是呼叫端的**指標值**（例如配置後回傳新位址），參數得是 `int **`。

```c
void alloc(int **out) { *out = malloc(sizeof(int)); }
```

## 常見誤解 / 陷阱

- **解參考 NULL、越界指標、已 free 的指標**都是未定義行為。
- **Double free**（同一塊 free 兩次）會破壞配置器狀態。
- **回傳區域變數的位址** → 函式結束那塊就失效，形成懸空指標。
- 指標加減**以元素為單位**，不是 byte。
- 隨意做型別轉換再讀寫，可能違反 aliasing / alignment 規則而出錯。

## 寫題時的檢查

- 這塊記憶體**誰配置、誰釋放**？
- 指標現在**還指著有效的東西嗎**？
- `free` 之前，需要的 `next` / 其他指標**先存起來了嗎**？
- 轉型後的讀寫**符合對齊與 aliasing** 規則嗎？
