# C 語言基礎

## C 是什麼樣的語言？

C 很「貼近硬體」：你直接面對記憶體位址、型別大小、變數的生死。
威力大、但**沒有安全網**——寫錯了不會幫你擋，常常變成「未定義行為」（程式可能崩潰、也可能看似正常）。

## 變數、型別與記憶體

每個變數佔一塊固定大小的記憶體，型別決定它佔幾個 byte、怎麼解讀。

```c
int   a = 65;     // 通常 4 byte 的整數
char  c = 'A';    // 1 byte，其實就是數字 65
double d = 3.14;  // 通常 8 byte 浮點數
```

## 陣列：從 0 開始數

```c
int a[5];   // 合法索引是 0,1,2,3,4（不是 1~5！）
            // a[5] 已經越界 → 未定義行為
```

陣列在記憶體裡是**連續排列**的一排格子，索引就是「從頭算第幾格」。

## 字串 = 以 '\0' 結尾的 char 陣列

C 沒有獨立的字串型別，字串就是一排 char，**結尾放一個 '\0'** 當終止符。

```c
char s[] = "cat";
// 實際內容： 'c' 'a' 't' '\0'   → sizeof(s) == 4（含結尾）
```

很多函式靠 '\0' 知道字串到哪結束；少了它就會一路讀過頭。

## 參數是「傳值」：函式拿到的是副本

```c
void addOne(int x) { x = x + 1; }   // 改的是副本，呼叫端不變

void addOnePtr(int *x) { *x = *x + 1; }  // 傳位址，才改得到本尊
```

想讓函式改到呼叫端的變數，必須**把位址傳進去**。這是後面指標章的核心動機。

## struct：把相關資料綁成一包

```c
struct Point { int x; int y; };
struct Point p = {3, 4};
```

注意：為了對齊 (alignment)，編譯器可能在欄位間塞 **padding**，
所以 `sizeof(struct)` **不一定**等於各欄位大小相加。

## static 區域變數：跨呼叫存活

```c
int counter(void) {
    static int n = 0;   // 只初始化一次，值會保留
    return ++n;         // 第一次回 1、第二次回 2…
}
```

一般區域變數每次進函式重生；`static` 的則活到程式結束。

## 常見誤解 / 陷阱

- **改寫字串字面值**（`char *s="cat"; s[0]='b';`）是未定義行為——字面值可能放在唯讀區。
- **有號整數溢位**是未定義行為，**別假設**它會「繞回去」。
- `malloc` 給的記憶體**沒初始化**（是垃圾值），讀之前要先寫，或改用會清零的 `calloc`。
- 陣列當參數會**退化成指標**，此時 `sizeof` 量到的是指標大小，不是整個陣列。

## 寫題時的檢查

- 每個陣列存取都在 `0 ~ n-1` 內嗎？
- 字串有沒有 '\0' 結尾？
- 函式真的改得到呼叫端資料嗎（有沒有傳位址）？
- 讀輸入是否用**有長度上限**的 API（如 `fgets` 而非 `gets`）？
