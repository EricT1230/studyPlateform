# Engineering Communication

## 核心概念
職場英文重點不是華麗，而是具體、負責、可協作。好的訊息會說明情境、影響、已做行動與下一步，避免指責或模糊抱怨。

## 解題重點
Code review 可用：`This might fail when the list is empty. Could we add a guard?` 先指出情境，再提出可行建議。Standup 可用：`I'm blocked on the payment API returning 500s; I opened a ticket and will pair with Sara.` 交代阻塞、行動與協助方式。

## 常見陷阱
`This is wrong.` 太籠統；`Did you even test this?` 帶攻擊性；`Still working on it.` 缺少進度資訊。不同意時先承認合理點，再用證據說明取捨。

## 練習前檢查
看選項是否具體、尊重、可執行。若句子包含 blame、絕對化或沒有下一步，通常不是最佳答案。
