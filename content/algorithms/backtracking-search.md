# 回溯搜尋 (Backtracking)

## 這是什麼？

回溯就是「**試一步、不行就退回來換一步**」的窮舉法。

生活類比：走迷宮。遇到岔路選一條走，走到死路就退回剛剛的岔路、改走另一條。一直退、一直試，直到走出去。

固定三個動作：**選擇 → 遞迴往下 → 取消選擇（退回）**。

```
search(目前狀態):
    若已完成 → 記錄答案
    對每個可能的下一步:
        若這步合法:
            做這步
            search(新狀態)     ← 遞迴往下
            取消這步           ← 回溯！把狀態還原
```

那個「取消這步」是回溯的靈魂。少了它，分支之間會互相污染。

## 解空間是一棵「選擇樹」

每一層做一個決定，整體展開成一棵樹。回溯就是在這棵樹上做深度優先走訪：

```
                (開始)
            /      |      \
          選A     選B     選C
          / \             /  \
        選B  選C        選A  選B
        ...  ...
```

走到底（完成）就記錄；走到死路（違規）就退回上一層換下一個。

## 實例：N 皇后

要在棋盤放皇后，彼此不能同行、同列、同對角線。一列一列放：

```
第1列放 (行0)：     Q . . .
第2列試 (行0)：     被同列攻擊 ✗ → 換
第2列試 (行1)：     被對角線攻擊 ✗ → 換
第2列放 (行2)：     Q . . .
                    . . Q .   ✓ 合法，往下一列
...若某列「每個位置都被攻擊」→ 退回上一列，移動上一個皇后
```

這個「整列都放不下就退回上一列重放」就是回溯。

## 剪枝：別走注定失敗的路

關鍵優化：一發現「目前的半成品**不可能**成功」，就立刻放棄這整條分支，不必走到底。

```
半成品已經違規 → 整棵子樹砍掉，直接退回
```

剪枝能讓實際跑得快很多，但**不會改變最壞複雜度**——回溯本質是指數級。

## 複雜度心裡有數

- 所有子集 (subsets)：每個元素選或不選 → 2ⁿ。
- 所有排列 (permutations)：n!。

剪枝只是減少實測分支數，不代表變成多項式。

## 常見誤解

- **忘記 undo（取消選擇）** → 上一條路的狀態污染下一條，答案全錯。
- 有重複元素卻沒處理 → 同一個答案被算很多次。解法：先排序，同一層跳過相同值，或用 start index。
- 把回溯跟 BFS / Dijkstra / A* 混在一起——後者有「代價/啟發值」的概念，回溯通常只是純窮舉。

## 解題時的判斷

- 要「列出所有解」或「找一個可行解」、且解可以一步步建構 → 回溯。
- 想清楚三件事：每層在**選什麼**、何時算**完成**、什麼情況**違規**。
- 有沒有可以**提早判定失敗**的條件？（剪枝）
- 答案是否在意順序、是否允許重複？
