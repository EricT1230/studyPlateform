# 複雜度與動態規劃 (Complexity & DP)

## 複雜度在講什麼？

複雜度是「當輸入變大，時間/空間會怎麼成長」。不在乎實際秒數，只在乎**成長的趨勢**。

規則：只留**最高次項**、丟掉常數。

```
3n + 10      → O(n)        （n 主導）
n² + 100n    → O(n²)       （n² 主導）
5            → O(1)        （不隨 n 變）
```

直覺感受成長差距（n 變大時）：

```
O(1)      ▏
O(log n)  ▎
O(n)      ▍▍▍▍
O(n log n)▊▊▊▊▊▊
O(n²)     █████████████████
O(2ⁿ)     █████████████████████████████…（爆炸）
```

三種 bound 別搞混：**worst-case（最壞）**、**average/expected（平均）**、**amortized（攤平）**。例如動態陣列 append 是攤平 O(1)，但單次擴容是 O(n)。

## 動態規劃 (DP)：把算過的記起來

DP 適用於兩個條件同時成立：
1. **重疊子問題**：同一個小問題被重複計算很多次。
2. **最佳子結構**：大問題的最佳解由子問題的最佳解組成。

看費氏數列 fib(5) 的遞迴——同一個值被算了好多次：

```
            fib(5)
          /        \
      fib(4)       fib(3)
      /    \        /    \
   fib(3) fib(2) fib(2) fib(1)     ← fib(3)、fib(2) 被重複計算
```

純遞迴是 O(2ⁿ)。把算過的存起來（memoization），每個值只算一次 → 變 O(n)。

## DP 的兩種寫法

- **由上而下 (memoization)**：照遞迴寫，但把結果快取，遇到算過的直接拿。
- **由下而上 (tabulation)**：開一張表，從最小的子問題往大填。

## 實例：DP 表怎麼填（湊錢幣）

用面額 {1, 3, 4} 湊出金額 6，最少要幾枚？
`dp[x]` = 湊出 x 的最少枚數，從小填到大：

```
金額 x:   0   1   2   3   4   5   6
dp[x]:    0   1   2   1   1   2   2
                              ▲
   dp[6] = 1 + min(dp[6-1], dp[6-3], dp[6-4])
         = 1 + min(dp[5], dp[3], dp[2])
         = 1 + min(2, 1, 2) = 2     → 用 3+3（兩枚）
```

每格只算一次、每次看固定幾個前面的格子 → 高效。
（注意：這題用貪心「先拿最大面額 4」會得 4+1+1=3 枚，比 DP 的 2 枚差——所以湊錢要用 DP，不是貪心。）

## 寫 DP 題的固定四步

1. **定義狀態**：`dp[...]` 到底代表什麼？（講清楚意義最重要）
2. **轉移式**：`dp` 怎麼由更小的 `dp` 算出來？
3. **初始化**：最小的 base case 是什麼？
4. **計算順序**：要先算好哪些格，才能算這格？

## 常見誤解

- 用 `O(n²)` 去「估」一個其實是 `Θ(n)` 的演算法——上界對但不精準。
- 0/1 背包**同一物品不能重複拿**（無限背包才可以）。
- DP 沒想清楚「計算順序」，用到還沒算好的格子 → 答案錯。
- 以為能貪心的都能貪心——湊錢、0/1 背包就是貪心會錯、要用 DP 的經典反例。

## 解題時的判斷

- 有沒有「重複在算同一個子問題」？→ 強烈暗示 DP。
- 狀態能不能用幾個變數唯一描述、而且可重用？
- 先想清楚狀態與轉移，再決定 memoization 還是 tabulation。
- 不確定貪心對不對 → 用 DP 比較保險。
