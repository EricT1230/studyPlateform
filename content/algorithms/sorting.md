# 排序 (Sorting)

## 這是什麼？

排序就是把一堆資料**按大小排好**。聽起來簡單，但「怎麼排」有很多種方法，各有取捨——這正是考試愛考的地方。

比較排序法時，看四件事：
- **時間**：快不快（平均 vs 最壞）。
- **空間**：要不要額外記憶體。
- **穩定性 (stable)**：相同值的東西，排完後相對順序有沒有變。
- **是不是用「比較」**：靠兩兩比大小，還是用別的招。

## 為什麼比較排序逃不過 O(n log n)

只要你的排序「靠兩兩比較」來決定順序，最壞情況就**不可能快過 O(n log n)**。

直覺：n 個東西有 n! 種可能排法。每次比較只能把可能性砍一半（像二十問）。要從 n! 種可能縮到 1 種，至少要問 log₂(n!) 次，而 log₂(n!) ≈ n log n。

```
要區分的可能順序：n! 種
每次比較的資訊量：1 bit（砍一半）
所以至少需要：log₂(n!) = Ω(n log n) 次比較
```

## 合併排序 (Merge Sort)：分了再合

策略：把陣列**對半切到不能再切**，再**兩兩合併**成有序的。

![合併排序：先把陣列一層層對半切開，再一層層合併成有序](https://commons.wikimedia.org/wiki/Special:FilePath/Merge%20sort%20algorithm%20diagram.svg "圖片來源：Wikimedia Commons「Merge sort algorithm diagram.svg」，公共領域 (Public Domain)")

用 [38, 27, 43, 3] 走一遍：

```
切：   [38, 27, 43, 3]
       /            \
   [38, 27]      [43, 3]
    /   \         /   \
  [38] [27]    [43]  [3]

合：  [27,38]      [3,43]
        \           /
        [3, 27, 38, 43]
```

切 log n 層、每層合併共 O(n)，所以是 O(n log n)。穩定，但需要額外 O(n) 空間。

## 三大 O(n log n) 排序的取捨（考點）

| 演算法 | 平均 | 最壞 | 空間 | 穩定 |
|---|---|---|---|---|
| 合併 Merge | n log n | n log n | O(n) | ✅ |
| 快速 Quick | n log n | **n²** | O(log n) | ❌ |
| 堆積 Heap | n log n | n log n | O(1) | ❌ |

- **快速排序**平均最快、常數小，但選到爛 pivot 時會退化成 O(n²)。
- **堆積排序**最壞也保證 n log n、又省空間，但不穩定、實測常數較大。
- **合併排序**穩定、最壞也 n log n，但要額外空間。

## 跳出 O(n log n)：非比較排序

如果資料是「範圍不大的整數」，可以不靠比較：

- **計數排序 (counting sort)**：開一個桶數陣列數每個值出現幾次，再依序倒出。O(n + k)（k = 值的範圍）。
- **基數排序 (radix sort)**：一位數一位數排。

它們突破 n log n 下界，因為**根本沒在做比較**（下界只管比較排序）。

## 常見誤解

- 把「平均」、「期望」、「最壞」混用。快速排序平均 O(n log n)，但最壞是 O(n²)。
- 以為「in-place」是指不動到輸入陣列——其實是指**不用額外空間**。
- selection sort、quicksort 通常**不穩定**。

## 解題時的判斷

- 要求**穩定** → merge sort（或穩定版）。
- 要**最壞保證 + 省空間** → heapsort。
- 一般情況要**最快** → quicksort（注意 pivot）。
- 資料是**小範圍整數** → counting / radix，可到線性。
- 資料**已近乎有序** → insertion sort 反而很快（接近 O(n)）。
