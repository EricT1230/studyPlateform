# 分治法 (Divide and Conquer)

## 這是什麼？

分治法就是「大問題拆成小問題，各自解決，再把答案拼回來」。

生活類比：要數一整疊考卷有幾份，一個人數很久。改成分一半給同事、各自數、最後加起來——這就是分治。

固定三步驟：
1. **Divide（分）**：把問題切成幾個較小的同類子問題。
2. **Conquer（治）**：遞迴解每個子問題。
3. **Combine（合）**：把子答案合併成原問題的答案。

## 最簡單的例子：二分搜尋

在已排序陣列找一個值，每次都看中間、砍掉一半：

```
找 5：  [1, 3, 5, 7, 9, 11, 13]
         mid=7，5<7 → 只看左半
        [1, 3, 5]
         mid=3，5>3 → 只看右半
              [5]  → 找到
```

每一步把範圍砍半，所以是 O(log n)。這就是「分」之後**只需處理一半**的威力。

## 用遞迴式描述成本

分治的成本常寫成這個式子：

```
T(n) = a · T(n/b) + f(n)
        ▲      ▲       ▲
   切成a個  每個n/b大  分割+合併的成本
```

合併排序就是 `T(n) = 2T(n/2) + Θ(n)`：切成 2 個一半大的、合併要 Θ(n)。

## 遞迴樹：一眼看出總成本

把每層的工作量畫出來、加總：

```
層              每層總工作
n               ─────────── n
n/2  n/2        ─────────── n      （2 × n/2）
n/4 n/4 n/4 n/4 ─────────── n      （4 × n/4）
...                          ...
共 log n 層，每層 n  →  總共 n log n
```

樹高是 log n（一直砍半），每層加起來都是 n，所以合併排序是 O(n log n)。**會畫遞迴樹，多數分治複雜度都能直接讀出來。**

## 哪些是分治

二分搜尋、合併排序、快速排序、quickselect（找第 k 小）、計算逆序對、最近點對、大數乘法 (Karatsuba)、矩陣乘法 (Strassen)。

## 常見誤解

- **忘了 base case** → 無限遞迴。每個分治都要有「小到可以直接回答」的終止條件。
- **合併時漏掉「跨兩半」的情況**。例如最近點對，最近的兩點可能一個在左半、一個在右半，combine 一定要檢查跨界。
- 遞迴式的 `f(n)` 到底是「分割成本」還是「合併成本」要分清楚——是兩者**合計**。

## 解題時的判斷

- 問題能不能切成「同類型、更小、且互相獨立」的子問題？→ 適合分治。
- 每次有沒有**真的縮小**問題規模？（不然不會收斂）
- 合併步驟有沒有涵蓋跨子問題的答案？
- 規則的 `aT(n/b)+f(n)` → 用 Master Theorem 直接套複雜度。
