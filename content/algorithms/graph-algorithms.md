# 圖演算法 (Graph Algorithms)

## 這一章在處理什麼？

圖描述「誰連到誰」。圖演算法就是在圖上回答問題：走得到嗎？最短路是哪條？怎麼連最省？

先看一張小圖（之後都用它）：

```
        A
       / \
      B   C
      |   |
      D---E
```

開始解任何圖題前，先確認三件事：
1. **有向還是無向**？
2. **有沒有權重**（邊有沒有成本）？
3. **權重會不會是負的**？

這三點直接決定你能用哪個演算法。

## BFS：一層一層往外擴

廣度優先從起點開始，先走完「距離 1」的鄰居，再走「距離 2」…像水波擴散。

從 A 出發：

```
第0層: A
第1層: B, C        （A 的鄰居）
第2層: D, E        （B、C 的鄰居）

走訪順序: A → B → C → D → E
```

關鍵用途：**無權重圖的最短路**。因為一層層擴，第一次碰到某點時，那就是最少步數。用 queue（佇列）實作。

## DFS：一條路走到底再退

深度優先選一條路一直往下，走到底再退回換下一條（就是回溯）。用 stack 或遞迴實作。

```
A → B → D → E → C   （一路深入，沒路了才回頭）
```

關鍵用途：偵測環、拓撲排序、找強連通元件 (SCC)。
注意：判斷「有向圖有沒有環」要看**遞迴堆疊**裡有沒有重複，不是只看 visited。

## 最短路徑：看權重選工具（最重要的考點）

| 情況 | 用什麼 | 複雜度 |
|---|---|---|
| 無權重 | **BFS** | O(V+E) |
| 非負權重 | **Dijkstra** | O((V+E) log V) |
| 有負權重 | **Bellman-Ford**（還能偵測負環） | O(V·E) |
| 所有點對之間 | **Floyd-Warshall** | O(V³) |

重點：**Dijkstra 不能用在有負邊的圖**——它的貪心假設「已確定的點不會再變短」，負邊會打破這個假設。

## 最小生成樹 (MST)：用最少成本連起所有點

把所有頂點連通、且邊權總和最小。兩個經典貪心：
- **Kruskal**：把邊由小到大加，只要不形成環就收（用並查集判環）。
- **Prim**：從一個點長出去，每次加「接到目前樹上最便宜的邊」。

注意：**MST ≠ 最短路徑**。MST 求的是「整體連通最省」，不是「兩點間最短」。

## 常見誤解

- 把 **MST 當最短路徑**（兩者目標不同）。
- 有負邊還用 **Dijkstra**（要用 Bellman-Ford）。
- 拓撲排序用在有環的圖（**只有 DAG 能拓撲排序**；Kahn 演算法若有點排不進去，代表有環）。
- 稀疏圖用鄰接矩陣 → 浪費 O(V²) 空間。

## 解題時的判斷

- 題目問的是**路徑、連通性、排序、切割，還是流量**？
- 圖**有向/無向、有無權重、有無負邊**？（決定演算法）
- 圖是**稀疏還是稠密**？（決定用鄰接串列還是矩陣）
- 要**最短路**還是**最省連通**？（前者最短路演算法、後者 MST）
