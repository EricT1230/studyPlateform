# 貪心法 (Greedy)

## 這是什麼？

貪心法就是「每一步都選**當下看起來最好**的，不回頭、不重來」。

生活類比：找零錢時，先拿最大面額、再拿次大…直到湊滿。每一步都貪最大的那枚。

它快又簡單，但有個大前提：**局部最好，必須真的能拼出全域最好。** 這不是理所當然——很多時候貪心會錯。

## 一個會成功的例子：活動選擇

有一堆活動，時間會重疊，一個人最多能參加幾個不衝突的？
**貪心規則：每次都選「最早結束」的那個。**

```
活動:   A ───────●
        B   ●─────────────●
        C        ●───●
        D              ●──────●
時間:   0   1   2   3   4   5   6   7

選 A（最早結束，t=2）→ 排除跟它重疊的 B
下一個最早結束又不衝突的 → C（t=4）
再下一個 → D（t=7）
結果：A, C, D，共 3 個
```

為什麼「最早結束」是對的？因為越早結束，留給後面的時間越多，不會比別的選法差。

## 貪心要成立的兩個條件

1. **貪心選擇性質 (greedy-choice property)**：每一步的局部最佳選擇，一定能成為某個全域最佳解的一部分。
2. **最佳子結構 (optimal substructure)**：原問題的最佳解，包含子問題的最佳解。

證明常用「交換論證 (exchange argument)」：拿任何一個最佳解，把它的選擇換成貪心的選擇，證明答案不會變差。

## 貪心會失敗的例子：0/1 背包

物品不能切，要在重量上限內裝最高總價值。
若貪心地「先拿 CP 值 (價值/重量) 最高的」——會錯：

```
上限 10kg
物品X: 6kg, $60   (CP=10)   ← 貪心先拿
物品Y: 5kg, $45   (CP=9)
物品Z: 5kg, $45   (CP=9)

貪心: 拿 X(6kg) → 剩 4kg → Y、Z 都塞不下 → 總值 $60
最佳: 拿 Y + Z (10kg)            → 總值 $90  ✗ 貪心輸了
```

（但「分數背包」物品**可以切**時，貪心拿 CP 值最高的就對了。差別就在能不能切。）

## 哪些是貪心

活動選擇 / 區間排程、Huffman 編碼、Kruskal 與 Prim（最小生成樹）、Dijkstra（非負權最短路）、分數背包。

## 常見誤解

- **0/1 背包不能用 CP 值貪心**（要用 DP）。
- 硬幣找零的貪心**只在特定幣制**（如台幣、美金）才對；任意幣制可能失敗。
- Dijkstra 的貪心**依賴非負權重**；有負邊就會錯（要用 Bellman-Ford）。
- set cover 等問題的貪心通常只是**近似解**，不是最佳。

## 解題時的判斷

- 先寫出你的「排序/選擇規則」，再問：**能不能舉出反例推翻它？**
- 能不能用交換論證證明貪心選擇安全？
- 題目要的是**最佳解**還是**可接受的近似**？（近似題貪心常夠用）
- 不確定貪心對不對 → 退一步想 DP。
