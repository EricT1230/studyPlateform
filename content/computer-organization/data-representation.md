# 資料表示法 (Data Representation)

## 核心觀念：一串 bit，由「誰解讀」決定意義

同樣 8 個位元 `1111 1111`：
- 當 **unsigned**（無號）→ 255
- 當 **two's complement**（二補數有號）→ −1

所以做題第一步永遠是問：**這串 bit 要當成什麼來看？**

## 二進位 ↔ 十六進位

十六進位每一位 = 剛好 4 個 bit，所以拿來「壓縮表示」位元最方便：

```
0011 1010
└─3─┘└─A─┘   →  0x3A  =  58（十進位）
```

一個 byte（8 bit）= 兩個十六進位數字。

## 二補數：電腦怎麼表示負數

規則：**最高位 (MSB) 代表「負的權重」**，其餘是正權重。

n 位元的範圍：`−2^(n−1)` 到 `2^(n−1)−1`（負的比正的多一個）。
4-bit 例子：範圍 −8 ~ +7。

求 −5 的二補數（4-bit）：
```
 5      = 0101
 取反   = 1010
 加 1   = 1011   →  −5
驗證：1011 = −8 + 0 + 2 + 1 = −5 ✓
```

好處：加減法電路「不用區分正負」，直接做就對（這就是為什麼用二補數而非 sign-magnitude）。

## 擴展位元數：零擴展 vs 符號擴展

把短的擴成長的，有號數要**複製符號位**，不是補 0：

```
−5 的 4-bit：    1011
符號擴展成 8-bit：1111 1011   ← 補符號位 1，值仍是 −5 ✓
若錯補 0：       0000 1011   ← 變成 +11，錯！
```

無號數才是零擴展（補 0）。

## 溢位：carry ≠ signed overflow

- **carry（進位）**：無號運算超出範圍。
- **signed overflow（有號溢位）**：兩個**同號**相加，結果卻變成**異號**。

```
4-bit 有號：  7 + 1 = 0111 + 0001 = 1000 = −8   ✗ 正+正變負 → 溢位
```

兩者是不同概念，看的是不同旗標。

## 位元組順序 (Endianness)

多 byte 的數值，存進記憶體時 byte 的擺法：

```
數值 0x12345678 存到位址 1000：
 little-endian：78 56 34 12   （低位 byte 放低位址）
 big-endian：   12 34 56 78   （高位 byte 放低位址）
```

注意：endianness 只改「byte 在記憶體的順序」，**不改數值本身**。

## 浮點數（一句話）

用 `符號 | 偏移指數 | 尾數` 來**近似**實數。因為是近似，`0.1` 這種十進位小數**無法被二進位精確表示**（所以 0.1+0.2 ≠ 0.3）。

## 常見誤解

- 符號擴展要**補符號位**，不是補 0。
- **carry 和 signed overflow 不是同一件事**。
- 浮點誤差是「二進位近似」造成，不是程式 bug。
- struct padding 來自**對齊 (alignment)**，不是資料被改寫。
- endianness 不改變數值，只改記憶體 byte 排列。

## 解題時的判斷

- 先確認：signed / unsigned？幾位元？
- 轉型題：先寫出原始 bit，再決定零擴展、符號擴展或重新解讀。
- 溢位題：無號看 carry、有號看「同號相加變異號」。
- 不要只算數值，**要追蹤 bit pattern**。
