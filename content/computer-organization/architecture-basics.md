# 計算機架構基礎

## 兩個層次：ISA vs 微架構

這章最重要的分界：

- **ISA（指令集架構）**：軟體**看得見**的契約——有哪些指令、暫存器、定址模式、例外行為。像 x86、ARM、RISC-V。
- **微架構 (microarchitecture)**：硬體**怎麼實作**這份契約——管線、ALU、快取、分支預測。

同一個 ISA 可以有很多種微架構實作（不同效能、功耗、價格）。
類比：ISA 是「考試規則」，微架構是「你怎麼準備考試」。

## CPU 在做什麼：取指 → 解碼 → 執行

CPU 不斷重複一個循環：

```
 ┌─────────────────────────────────────────┐
 ▼                                          │
取指 (Fetch) → 解碼 (Decode) → 執行 (Execute) → 寫回 (Write-back)
 從記憶體      看是什麼指令     ALU 算         結果存回暫存器/記憶體
```

PC（程式計數器）指向下一條指令；每跑完一條，PC 前進。

## 效能公式（一定要會代）

```
CPU 時間 = 指令數 × CPI ÷ 時脈頻率
           (I)      (每指令週期數)   (clock rate)
```

- 三個因子互相影響：減指令數、降 CPI、升頻率，都能加速。
- 單位先統一：1 GHz = 每秒 10⁹ 個 cycle。

陷阱：**CPI 低 ≠ 一定快**。還要看指令數和時脈（三者一起看）。

## Amdahl 定律：別只看局部快多少

整體加速受「**不能加速的部分**」限制：

```
某部分佔比 80%、把它加速 4 倍：
整體加速 = 1 / ( 0.2 + 0.8/4 ) = 1 / 0.4 = 2.5 倍   （不是 4 倍！）
```

重點：就算把某段加速到無限快，整體最多也只快 `1 / 不可加速比例`。所以**先優化佔比最大的部分**。

## 記憶體階層：用局部性換速度

```
   快、貴、小   暫存器 → Cache → 主記憶體 → 磁碟   慢、便宜、大
```

之所以有效，靠**局部性**：
- **時間局部性**：剛用過的，很快會再用（放快取）。
- **空間局部性**：用了某位址，鄰近的也常被用（一次載一整塊 cache line）。

## 常見誤解

- **管線提高的是吞吐 (throughput)**，不保證單一指令更快（latency 不一定變短）。
- **CPI 低不一定執行快**（要連同指令數、時脈一起算）。
- **中斷 (interrupt)** 比輪詢省，但仍有「保存狀態 + 跑 handler」的成本。

## 解題時的判斷

- 先判斷題目在問「ISA（可見契約）」、「微架構（怎麼加速）」還是「效能（算時間）」。
- 算時間 → 套 CPU time 公式、先統一單位。
- 「整體加速」題 → 用 Amdahl，看可加速比例。
- cache 題 → 從時間/空間局部性切入。
