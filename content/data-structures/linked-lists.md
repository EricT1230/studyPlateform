# 鏈結串列 (Linked List)

## 這是什麼？

鏈結串列是「一串節點，每個節點記住下一個在哪」。

生活類比：尋寶遊戲。每張紙條寫著一段提示，加上「下一張紙條在哪裡」。紙條本身散落各處，但只要從第一張開始，就能一張接一張找下去。

和陣列最大的不同：**節點不需要連續排在一起**。它們靠「指標 (pointer)」互相指，串成一條線。

## 一張圖看懂

![單向鏈結串列：每個節點存資料，再用指標指向下一個節點](https://commons.wikimedia.org/wiki/Special:FilePath/Singly-linked-list.svg "圖片來源：Wikimedia Commons「Singly-linked-list.svg」，公共領域 (Public Domain)")

用文字表示同樣的東西：

```
head
  │
  ▼
┌────┬───┐   ┌────┬───┐   ┌────┬───┐
│ 10 │ ●─┼──▶│ 20 │ ●─┼──▶│ 30 │ ✕ │   ✕ = NULL（結尾）
└────┴───┘   └────┴───┘   └────┴───┘
```

每個節點有兩部分：資料 (value) 和指向下一個的指標 (next)。

```c
struct Node {
    int value;
    struct Node *next;
};
```

## 跟陣列差在哪

```
陣列：   連續記憶體，靠「索引算位址」
         [10][20][30][40]

串列：   散落記憶體，靠「指標串起來」
         [10]→ [30]→ [20]→ [40]→ NULL
```

陣列用索引能直接跳；串列只能從 head 沿著指標一個一個走。

## 為什麼存取慢、插刪快

**存取第 i 個：慢 O(n)。** 沒有索引公式可算，只能從 head 走 i 步。

**插入/刪除：只要握有位置，就快 O(1)。** 因為只改幾個指標、不用搬資料。

在 10 和 20 之間插入 15：

```
插入前:  [10] ●──────────▶ [20]

插入後:  [10] ●──▶ [15] ●──▶ [20]
              只動了兩條指標：
              10 的 next 指向 15、15 的 next 指向 20
```

對比陣列要把後面整排往後搬——串列完全不用。

## 常見變體

**雙向 (doubly)**：節點多一個 prev，能往回走。

```
NULL ◀─● [10] ●⇄● [20] ●⇄● [30] ●─▶ NULL
```

**環狀 (circular)**：最後一個的 next 指回 head，繞成一圈。

## 常見誤解

- 想在「尾端」O(1) 插入，卻只留了 head → 還是得走到尾，O(n)。要另外保留 tail 指標。
- 刪除節點前要**先存下 next**，不然改了指標就找不到後面了。
- C 語言 `free(node)` 之後再讀 `node->next` 是 dangling pointer，會出事。
- 邊界要小心：空串列、刪第一個、刪最後一個。用 dummy head（虛擬頭節點）可以少很多 if。

## 解題時的判斷

- 需要「用索引快速讀取」→ 用陣列，不要用串列。
- 需要「頻繁在中間插入/刪除」且已握有節點 → 串列勝。
- 用串列實作 queue，要同時留 head 和 tail，enqueue/dequeue 才都 O(1)。
