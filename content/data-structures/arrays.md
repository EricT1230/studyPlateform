# 陣列 (Array)

## 這是什麼？

陣列是「一排連續、編號的格子」，每個格子放一個資料。

生活類比：郵局牆上一整排連號的信箱。第 0 號、第 1 號、第 2 號…緊緊排在一起，你只要知道號碼，一眼就能走到那一格。

陣列的關鍵特性就兩個：
- 格子在記憶體裡是**連續排列**的。
- 每個格子**大小一樣**。

## 記憶體裡長怎樣

假設一個 int 佔 4 個位元組，陣列從位址 1000 開始：

```
索引:      0      1      2      3      4
        ┌──────┬──────┬──────┬──────┬──────┐
        │  10  │  20  │  30  │  40  │  50  │
        └──────┴──────┴──────┴──────┴──────┘
位址:   1000   1004   1008   1012   1016
```

注意位址是 1000、1004、1008…每格剛好差 4。

## 為什麼存取第 i 個是 O(1)

因為位址可以「直接算出來」，不用一格一格找：

```
第 i 格的位址 = 起始位址 + i × 每格大小
```

實例：要拿第 3 格 → 1000 + 3 × 4 = 1012，直接跳過去。
不管陣列多長，這個計算都是一步完成，所以是 O(1)。

這就是陣列最大的優點：**用索引隨機存取超快。**

## 插入和刪除為什麼慢

問題出在「連續」這個特性。中間要插入或刪除，後面的元素全部要搬位子。

在索引 1 插入 99：

```
插入前:  10 │ 20 │ 30 │ 40
                ↑ 想插在這
插入後:  10 │ 99 │ 20 │ 30 │ 40
              └─ 20、30、40 全部往右移一格
```

最差情況（插在最前面）要搬 n 個元素，所以是 O(n)。
刪除同理，後面的要往前補。

## 動態陣列：append 為什麼是「攤平 O(1)」

固定陣列滿了就不能再加。動態陣列（如 C++ vector、Python list）會在滿的時候**自動擴容**：開一塊更大的（通常兩倍）、把舊資料複製過去。

```
容量滿了 → 開兩倍大的新空間 → 複製舊元素 → 加新元素
```

複製是 O(n)，聽起來很貴。但因為容量是「成倍」成長，擴容很少發生，把成本平均到每次 append 後，仍是 O(1)，這叫**攤平 (amortized) O(1)**。

實例：容量 1→2→4→8→16…，加 16 個元素只擴容了 4 次。

## 常見誤解

- 陣列的「存取」O(1) 指的是**用索引**；如果是「搜尋某個值」要逐格比對，那是 O(n)。
- append 攤平是 O(1)，但**單一次**擴容那刻是 O(n)，不是每次都很快。
- 陣列大小固定（靜態語言）；動態陣列只是幫你自動擴容，底層仍是連續記憶體。

## 解題時的判斷

- 需要「用索引快速讀取」、資料量穩定 → 陣列很適合。
- 需要「頻繁在中間插入/刪除」→ 陣列會慢，考慮 linked list。
- 看到「攤平複雜度」字眼，先想動態陣列的擴容。
