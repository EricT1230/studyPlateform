# 陣列 (Arrays)

## 核心概念
陣列是一段連續記憶體，元素大小固定，因此第 `i` 個元素位址可用 `base + i * size` 直接算出。這讓 random access 是 O(1)，但插入或刪除若發生在中間，後方元素通常都要搬移。

## 解題重點
看到陣列題，先分清楚「存取」、「搜尋」、「插入刪除」是哪一種操作。未排序陣列搜尋通常是 O(n)；已排序陣列可 binary search 到 O(log n)，但插入仍可能 O(n)。Dynamic array 的 append 要看攤還成本：大多數 append 是 O(1)，偶爾擴容複製是 O(n)，平均攤還後仍是 O(1)。

## 常見陷阱
不要把 `a[i]` 的 O(1) 誤推成所有操作都是 O(1)。陣列快在位址計算，不代表搜尋快。另一個陷阱是混淆 worst-case 和 amortized：單次 append 可能因擴容變 O(n)，但長期平均是攤還 O(1)。
