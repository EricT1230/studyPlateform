# 搜尋與規劃

## 搜尋問題在解什麼？

很多 AI 問題本質都是「**從現在的狀態，找一條路到目標狀態**」——走迷宮、解拼圖、路徑規劃都是。
要描述一個搜尋問題，需要五樣東西：

```
狀態 (state)        現在長怎樣          例：迷宮裡的格子
初始狀態            從哪開始
動作 (actions)      能做什麼            例：上下左右走
轉移模型            做了動作會變怎樣
目標測試            到了沒？
路徑成本            這條路花多少         例：走幾步
```

## 把搜尋想成一棵展開的樹

從初始狀態開始，不斷「展開」可走的下一步，形成一棵樹。
**frontier（邊界）= 還沒展開、排隊等著探索的狀態**。不同演算法的差別，就在**frontier 用什麼順序取出**。

## 非知情搜尋（沒有提示）

```
BFS  廣度優先  一層一層找   → 找到「最淺」的解；邊成本都相同時即最少步
DFS  深度優先  一路走到底   → 省記憶體，但可能鑽進深路或繞圈
UCS  成本一致  總是先展開「累積成本最小」的 → 邊成本不同時找最低成本
```

```
BFS：先把同一層全看完    DFS：先衝到最底再回頭
   ●                       ●
  /|\                      |
 ● ● ●  ← 先這層            ●
                           |
                           ● ← 先衝到這
```

## 知情搜尋（有啟發式 h）

**啟發式 h(n)** = 對「從 n 還要多少成本到目標」的估計（如直線距離）。

```
Greedy best-first  只看 h     快，但不保證最低成本
A*                 看 g + h   g=已付成本, h=估計剩餘 → 平衡兩者
```

A* 的核心是 `f(n) = g(n) + h(n)`：既不像 UCS 只回頭看、也不像 greedy 只往前猜。

## 啟發式的兩個重要性質

```
可採納 (admissible)：h 從不高估真實成本  → A* 保證找到最佳解
一致 (consistent)：  h 滿足三角不等式    → 圖搜尋也穩，不需重開節點
```

直覺：只要「不吹牛、不高估」，A* 就不會錯過更好的解。

## 對抗搜尋（兩人對弈）

下棋這種「對手會跟你作對」的問題用 **minimax**：我要最大化、對手要最小化，逐層交替。
**Alpha-beta 剪枝**砍掉「不可能影響結果」的分支，**不改變答案**、只加速。

## 規劃 (planning)：用前置條件與效果描述動作

規劃把每個動作寫成：

```
動作：拿起(積木A)
  前置條件 (precondition)：手是空的、A 上面沒東西
  效果 (effect)：手拿著 A、桌上少了 A
```

系統據此從「現在的世界狀態」推演到「目標條件」。重點是**三者分清楚**，別混在一起。

## 常見誤解

- **Greedy 只看估計距離**，可能很快但**不保證最低成本**。
- **DFS** 可能陷入深路或無窮循環，要記錄走過的狀態 (explored set)。
- 用了 explored set 也別忘記**成本更新規則**（找到更便宜的路要更新）。
- 規劃題別把 **goal / 動作效果 / 前置條件**混成一團。

## 解題時的判斷

- 先問題目要什麼：**完整性**（一定找得到）、**最佳性**（一定最便宜）、還是**省記憶體**？
- 邊成本都一樣 → BFS 就最佳；不一樣 → 用 UCS / A*。
- 有啟發式先檢查 **admissible、consistent、tie-breaking**。
- 對弈題想 **minimax + alpha-beta**；CSP 題先找變數、domain、constraint 與剪枝。
