# 資訊安全基礎

## 核心概念
資訊安全的基本目標是 CIA：機密性防未授權讀取，完整性防未授權竄改，可用性確保合法使用者能取得服務。實務上還要搭配認證、授權、稽核與最小權限。安全設計常假設單一防線會失效，因此需要分層防禦與可追蹤紀錄。

## 解題重點
Hash 是單向摘要，加密可在有 key 時還原。密碼儲存應使用 salt 加上慢速 password hashing/KDF。SQL injection 的核心防線是參數化查詢；XSS 要依輸出情境編碼；CSRF 需 token、SameSite 或來源檢查。Buffer overflow 屬記憶體邊界問題。權限設計要區分 authentication 與 authorization。

## 常見陷阱
只做前端檢查不能算安全防護。Salt 不需要保密，但也不能取代 bcrypt、scrypt、Argon2id 等成本設計。HTTPS 能驗證連線對象並保護傳輸，不代表應用沒有漏洞。

## 練習前檢查
先分類攻擊目標：偷資料、改資料、冒身分、重放請求、耗盡資源或取得更高權限。再選對應控制，不要把加密、簽章、授權混用。題目若提到外洩，還要思考輪換、撤銷與事後稽核；可用性題常連到 DoS 與備援設計。
