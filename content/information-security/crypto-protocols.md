# 密碼協定

## 核心概念
密碼協定不是只把資料加密，而是把機密性、完整性、身分驗證、新鮮性與金鑰生命週期組合起來。加密保護內容，MAC/HMAC 驗證完整性與共享金鑰持有者，數位簽章可用公開金鑰驗證來源。

## 解題重點
Diffie-Hellman 能協議共享秘密，但匿名 DH 不能抵抗主動中間人，仍需憑證、簽章或 PSK 綁定身分。TLS 憑證要檢查鏈、期限、用途與網域。AEAD 同時提供加密與認證，但 nonce 在同一把 key 下不可重複。KDF 可從共享秘密導出不同方向與用途的子金鑰，降低金鑰重用風險。

## 常見陷阱
Hash 不是可解密的加密。Nonce 不只是放進封包，還要被 MAC 或簽章保護並由接收方拒絕重放。JWT 驗證不能盲信 header 的 `alg`，演算法與 key 應由伺服器設定決定。裸 RSA、重複 ECDSA nonce 與錯誤憑證 pinning 都是常見協定雷區。

## 練習前檢查
先問此協定防的是竊聽、竄改、冒充、重放還是降級攻擊。再確認金鑰是否分用途、握手參數是否被 transcript 綁定，以及驗證失敗時是否立刻停止處理明文。
