# 機器學習基礎

## 機器學習在做什麼？

傳統程式是「人寫規則」；機器學習是「給資料，讓機器自己找規則」。

```
傳統：  輸入 + 規則(人寫) → 答案
機器學習： 輸入 + 答案(資料) → 規則(機器學出來)
```

學到的「規則」就是一個**模型**：一個能從特徵預測目標的函數。

## 三大類學習

```
監督式 (supervised)    有標準答案（標籤）
  ├ 分類：輸出類別     例：這封信是不是垃圾郵件
  └ 迴歸：輸出數值     例：明天房價多少

非監督式 (unsupervised) 沒有標籤，自己找結構
  ├ 分群：把相似的聚在一起
  └ 降維：用更少維度表達

強化學習 (reinforcement) 靠「獎勵」學策略   例：下棋、自駕
```

## 模型怎麼「學」？三個零件

1. **模型**：一個帶參數的函數（如 `y = wx + b`）。
2. **損失函數 (loss)**：衡量「預測」和「答案」差多少，越小越好。
3. **梯度下降**：往「讓損失下降最快」的方向，一小步一小步調參數。

```
損失曲面像一座山谷，梯度下降 = 沿坡往下走到谷底

  loss
   \            每步往低處移動
    \___       ↘
        \_____ ● ← 參數慢慢逼近最佳
```

**學習率 (learning rate)** = 每步跨多大：
- 太大 → 一步跨過谷底，來回震盪甚至發散。
- 太小 → 慢吞吞，要很久。

## 三份資料，各有用途

```
訓練集 (train)       學參數（模型靠它調 w、b）
驗證集 (validation)  調超參數、選模型
測試集 (test)        最後才用一次，估計真實泛化能力
```

**超參數**（學習率、K 值、層數…）不是訓練自動學到的，要靠驗證集去試。

## 常見誤解

- **特徵 (feature) 和標籤 (label) 不能混**：標籤是要預測的答案，混進特徵就是作弊。
- **資料洩漏 (data leakage)**：在切分資料**之前**就做標準化或特徵選擇，會讓測試資訊偷跑進訓練 → 評估過度樂觀。
- **類別特徵直接編成 0,1,2,3** 會引入假的「大小順序」（紅<綠<藍？），通常要 one-hot。
- **沒有萬靈丹 (No Free Lunch)**：沒有單一演算法在所有問題上都最好。

## 學習時的判斷

- 先認任務類型：有沒有標籤？輸出是類別、數值，還是分群？
- 評估用的資料**真的是模型沒看過的**嗎？
- 上線時**拿得到一樣的特徵**嗎（不能用到「未來才知道」的資訊）？
- 訓練不動時先看：學習率、特徵尺度、損失函數選對沒。
