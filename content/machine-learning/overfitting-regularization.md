# 過擬合與正則化

## 過擬合 vs 欠擬合

```
欠擬合 (underfit)   模型太簡單，連訓練資料的規律都抓不到
剛剛好 (good fit)    抓到規律、又不被雜訊帶偏
過擬合 (overfit)    模型太複雜，連訓練資料的「雜訊」都背起來
```

直覺：過擬合像「死背考古題」——同一份考卷滿分，換新題就垮。

```
欠擬合          剛剛好           過擬合
  /             ～～～           ∿∿曲折硬穿每一點∿
直線穿不過點    平滑貼近趨勢      被雜訊牽著走
```

## 偏差–變異權衡 (bias–variance)

```
高偏差 (bias)    系統性偏掉 → 通常欠擬合（訓練、驗證都差）
高變異 (variance) 太敏感、隨資料亂跳 → 通常過擬合（訓練好、驗證差）
不可約雜訊        資料本身的隨機性，誰都消不掉
```

模型越複雜：偏差↓、變異↑。要找中間的甜蜜點。

## 從學習曲線看是哪一種

```
誤差
 │＼train
 │  ＼______          ← 訓練誤差一直很低
 │        validation  ← 驗證誤差卻高 ＝ 過擬合（高變異）
 └──────────────► 模型複雜度

兩條都高且接近 ＝ 欠擬合（高偏差）
```

## 對付過擬合（高變異）的工具

```
更多資料        最直接，雜訊被稀釋
降低模型複雜度  少一點參數/層數
正則化          L1 / L2，懲罰過大的權重
Dropout         訓練時隨機關掉部分神經元，逼網路不依賴單一路徑
Early stopping  驗證誤差開始回升就停
資料增強        旋轉/裁切影像等，等於變相增加資料
```

## L1 vs L2 正則化

在損失裡加一項「權重的懲罰」，逼模型別把權重養太大：

```
L2 (Ridge)  懲罰 Σwᵢ²   → 權重整體變小、平滑
L1 (Lasso)  懲罰 Σ|wᵢ|  → 把一些權重壓成 0 → 順便做特徵選擇
```

要稀疏 / 自動選特徵用 L1；要平滑、穩定用 L2。

## 集成方法的方向

```
Bagging（如隨機森林）  多模型平均 → 主要降「變異」
Boosting（如 XGBoost） 逐步修正錯誤 → 主要降「偏差」（但跑太久會反過來過擬合）
```

## 常見誤解

- **訓練誤差低 ≠ 模型好**，一定要看驗證/測試誤差。
- **正則化係數調太大**會矯枉過正，從過擬合變成欠擬合。
- **拿測試集反覆選模型**本身就是資料洩漏。
- **Boosting 輪數太多**也會過擬合，不是越久越好。

## 診斷時的判斷

- 先比**訓練 vs 驗證**：兩者都差→高偏差；訓練好驗證差→高變異。
- 高偏差 → 加容量、減正則化、做更多特徵。
- 高變異 → 加資料、加正則化/ Dropout、降複雜度。
- 看**曲線趨勢**比看單一分數更能判斷問題在哪。
