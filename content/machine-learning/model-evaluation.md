# 模型評估

## 核心概念
模型評估是在估計泛化能力，而不是回報訓練集成績。分類、迴歸與排序問題使用不同指標；真正要選哪個指標，取決於錯誤成本與產品目標。好的評估流程應保留最後一次獨立測試，避免把資料反覆用到模型選擇中。

## 解題重點
混淆矩陣包含 TP、FP、TN、FN。Accuracy 看整體正確率，但在類別不平衡時可能誤導。Precision 關心預測為正的可信度，recall 關心實際正類被抓到多少，F1 是兩者調和平均。調高門檻通常提高 precision、降低 recall。迴歸常看 MAE、MSE、RMSE、R²；MSE/RMSE 對大誤差更敏感，MAE 較穩健。

## 常見陷阱
ROC-AUC 在極端不平衡時可能過度樂觀，PR-AUC 常更貼近少數類問題。交叉驗證若沒有分層，少數類可能分布不均。反覆用測試集調參會污染最終評估。

## 練習前檢查
先問哪種錯誤代價最高：誤報還是漏報？資料是否不平衡？指標是否在獨立資料或交叉驗證上計算？若比較模型，也要確認切分方式、資料期間與前處理一致。
