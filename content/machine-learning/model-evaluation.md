# 模型評估

## 評估在量什麼？

不是看「模型在訓練資料考幾分」，而是估計**它面對沒看過的資料能多準**（泛化能力）。
模型把訓練題背起來很容易，重點是換一份新考卷還行不行。

## 混淆矩陣：四種情況

分類問題的一切指標都從這張表長出來：

```
                 實際為正    實際為負
預測為正         TP          FP（誤報，冤枉）
預測為負         FN（漏報）   TN
```

- **TP**：抓對的正例　**FP**：把負的誤判成正（誤報）
- **FN**：把正的漏掉（漏報）　**TN**：正確判為負

## 常用分類指標

```
Accuracy  = (TP+TN) / 全部        整體答對率
Precision = TP / (TP+FP)          說「是」的有多可信
Recall    = TP / (TP+FN)          真正的正例抓到多少
F1        = Precision、Recall 的調和平均
```

**直覺**：
- 在意「別冤枉人」（誤報代價高，如垃圾郵件誤刪）→ 重 **Precision**。
- 在意「別漏掉」（漏報代價高，如癌症篩檢）→ 重 **Recall**。

**Accuracy 在類別不平衡時會騙人**：99% 的人沒病，模型全猜「沒病」也有 99% accuracy，但 recall=0，毫無用處。

## 門檻會拉扯 Precision 與 Recall

調高「判為正」的門檻 → 出手更保守 → **Precision 上升、Recall 下降**（反之亦然）。兩者通常此消彼長。

## 迴歸指標

```
MAE  平均絕對誤差     對離群值較穩健
MSE  平均平方誤差     放大大誤差（平方）
RMSE MSE 開根號       單位回到原本尺度
R²   解釋了多少變異   1 最好、0 等於只猜平均
```

要「嚴懲大錯」用 MSE/RMSE；要「不被極端值帶偏」用 MAE。

## 交叉驗證：別把命運押在一次切分

把資料切成 K 份，輪流拿一份當驗證、其餘訓練，跑 K 次取平均——比單次切分更穩、更省資料。
類別不平衡時要用**分層 (stratified)**，確保每折的類別比例一致。

## 常見誤解

- **ROC-AUC 在極端不平衡時會過度樂觀**；少數類問題看 **PR-AUC** 更貼切。
- **反覆拿測試集調參** = 資料洩漏，最終分數會虛高。測試集只能用最後一次。
- 比較兩個模型時，**切分方式、資料期間、前處理必須一致**，否則比的不是模型本身。
- 高 accuracy ≠ 好模型，先確認資料平不平衡。

## 評估時的判斷

- 先問：**誤報還是漏報的代價高**？據此選 Precision / Recall 偏重。
- 資料**平衡嗎**？不平衡別只看 accuracy、ROC-AUC。
- 指標是在**獨立資料或交叉驗證**上算的嗎？
- 測試集是否**全程只用一次**？
